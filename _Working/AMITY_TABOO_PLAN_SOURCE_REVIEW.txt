AMITY TABOO — GAMEPLAY/UI BEHAVIOR SPEC v2 (WITH localStorage PERSISTENCE)

INSTRUCTION CARD VS INFORMATION CARD (TWO DIFFERENT THINGS)

1) INSTRUCTION CARD (DECK CONTROL SCREEN)
- Purpose:
  - Controls the lifecycle of a deck (build, ready, start play, reset).
- Rule:
  - If the instruction card is reached/triggered, behavior returns to square one.
  - A NEW deck is generated.
  - BEGIN PLAY is disabled until the new deck is fully built.
  - Any existing saved deck state is cleared (see PERSISTENCE).

2) INFORMATION CARD (ABOUT/HELP SCREEN)
- Purpose:
  - About the game, why it exists, Buy Me A Coffee link, etc.
- Rule:
  - Information card must NOT reset the deck.
  - Information card must provide a clear “BACK TO PLAY” control that returns the user to the exact card they were on.
  - The game resumes at the same card index and same deck instance.


STATE A — INSTRUCTION_LOADING (INSTRUCTION CARD VISIBLE, DECK BUILDING)

BACKGROUND (GAS)
- On entry to this state, Apps Script:
  - Reads ADMIN percentages (Easy/Medium/Hard).
  - Shuffles prompt rows.
  - Selects 100 cards according to the percentages.
  - Builds a 100-card deck.

UI
- Instruction card shows a working indicator in place of “Your deck is ready.”
  - Text: “L, o, a, d, i, n, g, ., ., .”
- Buttons:
  - BEGIN PLAY: grey (disabled look)
  - BACK: grey (disabled)
  - NEXT: grey (disabled)
- Status bar (not read by screen readers):
  - Deck: 0 of 0
  - Card ID: #0000

SCREEN READER
- Do NOT repeatedly announce “loading”.
- Announce once (polite, complete sentence):
  - “Building your deck. Please wait.”


STATE B — INSTRUCTION_READY (INSTRUCTION CARD VISIBLE, DECK BUILT)

UI
- “Loading…” disappears.
- “Your deck is ready.” appears.
- Buttons:
  - BEGIN PLAY: black (enabled)
  - NEXT: green (enabled)
  - BACK: grey (disabled; nothing to go back to yet)
- Status bar:
  - Deck: 0 of 100
  - Card ID: #0000

SCREEN READER
- Announce once (polite, complete sentence):
  - “Your deck is ready. You may begin playing.”


STATE C — PLAYING (CARD MODE)

ON BEGIN PLAY
- Instruction card disappears.
- Card 1 appears.
- Status bar:
  - Deck: 1 of 100
  - Card ID: sheet row number for that prompt

CARD LAYOUT
- Green header field:
  - Speak label: “Target word:” then the target word.
  - Display: target word in BLACK, ALL CAPS.
- White field:
  - Speak label: “Taboo words:” then the taboo list.
  - Display: taboo words in RED, ALL CAPS LIST.

NAVIGATION (BACK / NEXT)
- NEXT:
  - Advances to the next card.
  - Updates Deck: N of 100.
  - Updates Card ID to the current card’s sheet row number.
- BACK:
  - Goes to the previous card.
  - Updates Deck and Card ID accordingly.
- Special rule:
  - BACK on Card 1 returns to INSTRUCTION_LOADING AND STARTS A NEW DECK BUILD (square one).


END OF DECK

- When user is on Card 100 and presses NEXT:
  - Return to INSTRUCTION_LOADING.
  - Immediately begin building a NEW deck.
  - While building:
    - Deck: 0 of 0
    - Card ID: #0000
    - BEGIN PLAY disabled until deck is ready.


STATUS BAR RULES

- Status bar is NOT read by screen readers.
- Values:
  - Loading: Deck 0 of 0, Card ID #0000
  - Ready (before play): Deck 0 of 100, Card ID #0000
  - Playing: Deck N of 100, Card ID = sheet row number


PERSISTENCE / RESUME POLICY (CHOSEN: localStorage)

GOAL
- The game must survive reloads, tab closes, and device restarts.
- The user should return to the same deck and same card index after interruption.

RULES
1) Persist the active deck session in localStorage:
- Deck data (100 cards)
- Current card index (0..99)
- Deck ID (nonce/timestamp)
- Any metadata needed for debugging (optional)

2) On page load:
A) If a valid saved deck exists:
- Resume directly into PLAYING at the saved index (or show a brief resume-ready state if desired later).
B) If no valid saved deck exists:
- Enter INSTRUCTION_LOADING and build a new deck.

3) Reset behavior:
- Any entry to INSTRUCTION_LOADING that represents “start over / new deck” MUST first clear localStorage deck/session state.
- The INFORMATION CARD must NOT clear localStorage; it is non-destructive.

4) User expectations:
- The instruction card is the “new deck/reset” path.
- The information card is “about/help” and returns to play.


NOTES FOR FUTURE IMPLEMENTATION
- Prefer a schema version field for the stored object so future changes can invalidate older saved sessions safely.

AMITY TABOO — IMPLEMENTATION NOTES (FOR FUTURE AGENTS)

HIGH-LEVEL APPROACH
- Model the UI as a small finite-state machine with explicit states:
  A) INSTRUCTION_LOADING
  B) INSTRUCTION_READY
  C) PLAYING
  D) INFO_OVERLAY (non-destructive; returns to PLAYING)
- Keep one in-memory “session” object that owns:
  - deck array (100 cards)
  - currentIndex (0..99)
  - deckId (timestamp/nonce, optional)
  - isBuilding flag
- Prefer client-side deck persistence (sessionStorage/localStorage) so play can continue through pauses and brief network interruptions.


DATA MODEL (RECOMMENDED)
DeckCard object (one per card):
- index: 1..100 (position in deck)
- sheetRow: integer (row number for Card ID)
- target: string (ALL CAPS display)
- taboo: string[] (ALL CAPS display list)
- difficulty: "easy" | "medium" | "hard" (optional for debugging)

DeckSession object:
- deckId: string (e.g., ISO timestamp + random suffix)
- createdAt: number (ms)
- cards: DeckCard[100]
- currentIndex: number (0..99)
- adminMix: {easyPct, medPct, hardPct} (optional)
- sourceSheetId / tab name (optional)
- version: number (schema version for persistence)


UI STATE MACHINE (BEHAVIOR MAP)

STATE: INSTRUCTION_LOADING
- Visual:
  - Instruction card visible
  - Working indicator text shown instead of “Your deck is ready.”
  - BEGIN PLAY disabled (grey)
  - BACK disabled (grey)
  - NEXT disabled (grey)
  - Status bar: Deck 0 of 0, Card ID #0000
- Accessibility:
  - aria-live polite: “Building your deck. Please wait.” (announce once)
- Transition out:
  - When deck build promise resolves -> INSTRUCTION_READY

STATE: INSTRUCTION_READY
- Visual:
  - “Your deck is ready.” displayed
  - BEGIN PLAY enabled (black)
  - NEXT enabled (green)
  - BACK disabled (grey)
  - Status bar: Deck 0 of 100, Card ID #0000
- Accessibility:
  - aria-live polite: “Deck ready.” (announce once)
- Actions:
  - BEGIN PLAY -> PLAYING at currentIndex=0
  - NEXT -> PLAYING at currentIndex=0 (optional shortcut if desired; spec currently allows NEXT to be enabled here)
  - Any “reset/start new deck” action -> INSTRUCTION_LOADING and begins new build

STATE: PLAYING
- Visual:
  - Card view replaces instruction text
  - Header: TARGET WORD (black ALL CAPS)
  - Body: TABOO WORDS (red ALL CAPS list)
  - Buttons:
    - NEXT enabled (green) unless at end-of-deck rule requires otherwise
    - BACK enabled (green) except special rule below
  - Status bar:
    - Deck: (currentIndex+1) of 100
    - Card ID: sheetRow for current card
- Navigation rules:
  - NEXT on 1..99: currentIndex++
  - NEXT on 100:
    - transition to INSTRUCTION_LOADING and immediately start a NEW deck build
  - BACK on 2..100: currentIndex--
  - BACK on 1:
    - transition to INSTRUCTION_LOADING and immediately start a NEW deck build (“square one”)
- Accessibility:
  - The status bar should not be in the reading order.
  - Use visible labels “Target word:” and “Taboo words:” (or aria-label equivalents) so screen readers speak those prefixes.

STATE: INFO_OVERLAY (INFORMATION CARD)
- Purpose:
  - “About / why no-timer taboo / buy me a coffee” content
- Critical rule:
  - MUST NOT reset or rebuild the deck
- UI:
  - Present as an overlay/modal or a separate view that preserves DeckSession and currentIndex
  - Provide a large “BACK TO PLAY” control at the bottom
- Actions:
  - BACK TO PLAY returns to PLAYING with the same currentIndex and same deck


DECK BUILD IMPLEMENTATION OPTIONS

OPTION 1 (CURRENT SINGLE-PAGE APPS SCRIPT HTMLSERVICE)
- Build deck in Apps Script, send data to phone.html via templating or google.script.run callback.
- Note: google.script.run is only available when the UI is served by Apps Script.

OPTION 2 (FUTURE EXTERNAL HOSTING)
- Apps Script becomes API-only and returns JSON via ContentService.
- UI (hosted elsewhere) calls Apps Script endpoints to fetch a newly built deck.
- Prefer returning the full 100-card deck in one response so the UI can operate offline once loaded.


CLIENT-SIDE PERSISTENCE (RECOMMENDED)
- Save DeckSession to sessionStorage (default) or localStorage (optional).
- On page load:
  - If a valid DeckSession exists and is “fresh enough,” restore it and resume in PLAYING.
  - If not, start INSTRUCTION_LOADING and build a new deck.
- If you want “always new deck on refresh,” then ignore stored sessions by default and only use persistence to survive transient reloads during play (opt-in flag).


LOADING INDICATOR + SCREEN READER NOTES
- Do not update aria-live text repeatedly (it will be spoken repeatedly).
- Approach:
  - Visual: animated “L, o, a, d, i, n, g…” is fine.
  - Screen reader: one-time polite announcement on entry, and one-time “Deck ready.” on completion.


TEST CASES (MINIMUM)
1) Initial load -> loading -> ready -> begin play -> card 1.
2) NEXT through card 100 -> NEXT -> instruction loading (new deck) -> ready.
3) BACK on card 1 -> instruction loading (new deck) -> ready.
4) INFO overlay from mid-deck -> BACK TO PLAY returns to same card index and card ID.
5) Screen reader reads “Target word:” + target and “Taboo words:” + list; does not read status bar.
6) Network interruption after deck built: BACK/NEXT still work (if client-side deck persisted).


IMPLEMENTATION SAFETY
- Keep state transitions centralized (one function) to avoid drift between UI and logic.
- Keep button enable/disable derived from state and currentIndex (not ad-hoc toggles).

const ADMIN_SHEET_NAME = 'ADMIN';
const CARDS_SHEET_NAME = 'Cards';

function doGet() {
  // Default to phone-first UI unless explicitly overridden with ?ui=app
  const ui = null;
  const filename = (ui === 'app') ? 'app' : 'phone';
  const html = HtmlService.createHtmlOutputFromFile(filename)
    .setTitle('Amity Games Taboo')
    .setXFrameOptionsMode(HtmlService.XFrameOptionsMode.ALLOWALL);

  return html;
}

function include(filename) {
  return HtmlService.createHtmlOutputFromFile(filename).getContent();
}

/**
 * Returns one card at a given sheet row index (2-based; row 1 is header)
 * plus total card count.
 */
function getCard(row) {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const sheet = ss.getSheetByName(CARDS_SHEET_NAME);
  if (!sheet) {
    throw new Error(`Missing sheet: ${CARDS_SHEET_NAME}`);
  }

  const lastRow = sheet.getLastRow();
  const total = Math.max(0, lastRow - 1); // exclude header row
  if (total <= 0) {
    return { total: 0, target: '', taboo: [] };
  }

  const safeRow = Math.min(Math.max(2, Number(row) || 2), lastRow);
  const values = sheet.getRange(safeRow, 1, 1, sheet.getLastColumn()).getValues()[0];

  // Expected columns:
  // A: DIFFICULTY, B: TARGET, C..G: TABOO1..TABOO5, H: SOURCE (optional)
  const target = (values[1] ?? '').toString().trim();
  const taboo = [];
  for (let i = 2; i <= 6; i++) {
    const v = (values[i] ?? '').toString().trim();
    if (v) taboo.push(v);
  }

  return {
    total,
    target,
    taboo
  };
}


/**
 * Build a 100-card deck based on ADMIN difficulty percentages.
 * Returns:
 * {
 *   deckId: string,
 *   total: number, // always 100 when possible
 *   cards: [{ index: number, rowId: number, target: string, taboo: string[] }]
 * }
 */
function buildDeck() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();

  const admin = ss.getSheetByName(ADMIN_SHEET_NAME);
  const cardsSheet = ss.getSheetByName(CARDS_SHEET_NAME);
  if (!cardsSheet) throw new Error(`Missing sheet: ${CARDS_SHEET_NAME}`);

  const settings = _readAdminDifficultySettings_(admin);
  const easyPct = settings.easyPct;
  const mediumPct = settings.mediumPct;
  const hardPct = settings.hardPct;

  const lastRow = cardsSheet.getLastRow();
  const lastCol = cardsSheet.getLastColumn();
  if (lastRow < 2) return { deckId: _nonce_(), total: 0, cards: [] };

  const rows = cardsSheet.getRange(2, 1, lastRow - 1, lastCol).getValues();

  const easy = [];
  const medium = [];
  const hard = [];
  const all = [];

  for (let i = 0; i < rows.length; i++) {
    const r = rows[i];
    const difficulty = (r[0] ?? '').toString().trim().toLowerCase();
    const target = (r[1] ?? '').toString().trim();
    const taboo = [];
    for (let t = 2; t <= 6; t++) {
      const v = (r[t] ?? '').toString().trim();
      if (v) taboo.push(v);
    }
    if (!target) continue;

    const rowId = i + 2; // sheet row number

    const card = { rowId, target, taboo };

    all.push(card);

    if (difficulty.includes('easy')) easy.push(card);
    else if (difficulty.includes('med')) medium.push(card);
    else if (difficulty.includes('hard')) hard.push(card);
    else {
      // Unlabeled difficulty goes to general pool
    }
  }

  _shuffleInPlace_(easy);
  _shuffleInPlace_(medium);
  _shuffleInPlace_(hard);

  const targetTotal = 100;

  let easyN = Math.round(targetTotal * (easyPct / 100));
  let mediumN = Math.round(targetTotal * (mediumPct / 100));
  let hardN = targetTotal - easyN - mediumN;

  if (hardN < 0) { hardN = 0; mediumN = targetTotal - easyN; }
  if (mediumN < 0) { mediumN = 0; easyN = targetTotal; }

  const picked = [];
  function take(fromArr, n) {
    while (n > 0 && fromArr.length > 0) {
      picked.push(fromArr.shift());
      n--;
    }
    return n;
  }

  let remEasy = take(easy, easyN);
  let remMed = take(medium, mediumN);
  let remHard = take(hard, hardN);

  // Fill any shortfall from remaining pools, then from all
  const pool = [].concat(easy, medium, hard);
  _shuffleInPlace_(pool);

  let missing = remEasy + remMed + remHard;
  while (missing > 0 && pool.length > 0) {
    picked.push(pool.shift());
    missing--;
  }

  if (missing > 0) {
    _shuffleInPlace_(all);
    const seen = new Set(picked.map(c => c.rowId));
    for (let i = 0; i < all.length && missing > 0; i++) {
      const c = all[i];
      if (seen.has(c.rowId)) continue;
      picked.push(c);
      missing--;
    }
  }

  const deckId = _nonce_();
  const cards = picked.slice(0, targetTotal).map((c, idx) => ({
    index: idx + 1,
    rowId: c.rowId,
    target: c.target,
    taboo: c.taboo
  }));

  return { deckId, total: cards.length, cards };
}

/**
 * Reads ADMIN difficulty percentages with tolerant parsing.
 * Defaults to 34/33/33 if not found or invalid.
 */
function _readAdminDifficultySettings_(adminSheet) {
  const def = { easyPct: 34, mediumPct: 33, hardPct: 33 };
  if (!adminSheet) return def;

  const values = adminSheet.getDataRange().getValues();
  if (!values || values.length === 0) return def;

  // Strategy 1: key/value in first two columns
  const kv = {};
  for (let r = 0; r < values.length; r++) {
    const k = (values[r][0] ?? '').toString().trim().toLowerCase();
    const v = values[r][1];
    if (!k) continue;
    kv[k] = v;
  }

  function num(x) {
    const n = Number(String(x).replace('%','').trim());
    return Number.isFinite(n) ? n : NaN;
  }

  function findPct(name) {
    // look for any key containing name + percent-ish
    const keys = Object.keys(kv);
    for (const k of keys) {
      if (k.includes(name) && (k.includes('%') || k.includes('percent') || k.includes('pct'))) {
        const n = num(kv[k]);
        if (Number.isFinite(n) && n >= 0) return n;
      }
    }
    // look for exact name key
    for (const k of keys) {
      if (k === name || k.includes(name + ' ')) {
        const n = num(kv[k]);
        if (Number.isFinite(n) && n >= 0) return n;
      }
    }
    return NaN;
  }

  let easyPct = findPct('easy');
  let mediumPct = findPct('medium');
  let hardPct = findPct('hard');

  // Strategy 2: header row "Easy/Medium/Hard" with next row values
  if (![easyPct, mediumPct, hardPct].some(Number.isFinite)) {
    const header = values[0].map(x => (x ?? '').toString().trim().toLowerCase());
    const row1 = values[1] || [];
    function headerIdx(term) {
      return header.findIndex(h => h.includes(term));
    }
    const ei = headerIdx('easy');
    const mi = headerIdx('med');
    const hi = headerIdx('hard');
    if (ei >= 0) easyPct = num(row1[ei]);
    if (mi >= 0) mediumPct = num(row1[mi]);
    if (hi >= 0) hardPct = num(row1[hi]);
  }

  if (!Number.isFinite(easyPct)) easyPct = def.easyPct;
  if (!Number.isFinite(mediumPct)) mediumPct = def.mediumPct;
  if (!Number.isFinite(hardPct)) hardPct = def.hardPct;

  // Normalize if totals are off
  let sum = easyPct + mediumPct + hardPct;
  if (!Number.isFinite(sum) || sum <= 0) return def;

  easyPct = (easyPct / sum) * 100;
  mediumPct = (mediumPct / sum) * 100;
  hardPct = (hardPct / sum) * 100;

  return {
    easyPct: Math.round(easyPct),
    mediumPct: Math.round(mediumPct),
    hardPct: Math.max(0, 100 - Math.round(easyPct) - Math.round(mediumPct))
  };
}
