=======================================================
PROJECT OVERVIEW
=======================================================

PROJECT NAME: Amity Taboo  

Amity Taboo is a Google Apps Script + HTML web app that delivers an accessible, single-screen Taboo-style prompt experience.

KEY CHARACTERISTICS:  

1. Browser-based web app served from Google Apps Script.  

2. No teams.  
3. No timers.  
4. No scores.  

5. Card-based UI designed to fit the viewport without scrolling.  

6. Logic lives in Apps Script; UI lives in app.html.

=======================================================
CORE ARCHITECTURE
=======================================================

A. GOOGLE APPS SCRIPT PROJECT  
Houses server-side logic and HTML delivery.

B. CONTAINER SPREADSHEET (AUTHORITATIVE DATA SOURCE)  
Stores cards, settings, and deck state.

C. WEB APP  
Deployed from the Apps Script project and served to players.

These store generated decks and should not be edited manually.

=======================================================
UI & BEHAVIOR PRINCIPLES
=======================================================

1. Single, fixed viewport — no scrolling.  

2. Clean card layout: title, target word, taboo list, status bar.  

3. Begin Play button should only activate once a deck is built.  

4. Text must remain screen-reader friendly.  

5. Minimal visual chrome; focus on readability.

=======================================================
KNOWN ISSUES / NOTES
=======================================================

1. If “Begin Play” does not activate, check whether a deck was actually generated in hidden sheets.  

2. Deck generation depends on valid ADMIN percentages summing to 1.0.  

3. Do not manually edit _DECK_INDEX or _DECK_CARDS.
