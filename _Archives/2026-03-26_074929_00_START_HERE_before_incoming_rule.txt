LOCKSTEP MODE

Unless the user explicitly directs otherwise, work in lockstep.

Each cycle follows this order:

GOAL:
Concise statement of the next task.

ACTION:
Concise statement of the immediate next action.

Then one exchange only:

A. Agent gives one immediate step, command stack, file read, click path, or focused instruction.

B. Agent stops and waits for returned evidence.

C. User returns output, result, screenshot, or confirmation.

RESULT:
Concise statement of what the returned evidence shows, including success, failure, discoveries, and what informs the next GOAL.

WORKING RULES

1. Do not begin task work before startup reading is complete.

2. Do not combine multiple future branches before current evidence returns.

3. RESULT exists only after evidence exists.

4. Continue one observable step at a time unless the user requests broader execution.

PURPOSE

This keeps work observable, reduces assumption drift, and preserves handoff continuity.
